# Scalable Big Data Analytics Framework for Time-Domain Astronomy

## Overview
This project presents a **scalable big data analytics pipeline** for processing large-scale astronomical datasets using **Apache Spark** and **HDFS**. It focuses on handling high-volume FITS image data generated by modern sky surveys and extracting meaningful insights for **transient object detection**.

The system implements a **distributed, fault-tolerant architecture** combined with **machine learning techniques** to efficiently process and analyze astronomical data.

---

## Objectives
- Build a **distributed ingestion and storage system** for astronomical data  
- Design **parallel processing workflows** for multi-source telescope data  
- Implement **resource-aware computation strategies**  
- Develop **scalable pipelines** for future data growth  

---

## Tech Stack
- **Apache Spark (3.5.7)** – Distributed data processing  
- **HDFS** – Distributed storage  
- **Python (SciPy, Scikit-learn, Astropy)** – Scientific computation  
- **Parquet & JSON** – Data storage formats

## Dataset
- **Source**: Zwicky Transient Facility (ZTF)  
- **Volume**: 122,382 FITS files (~2.29 GB)

### Features
| Feature | Description |
|--------|------------|

| image_type | TEMPLATE / SCIENCE / DIFFERENCE |
| mean_intensity | Average pixel intensity |

| bright_object_count | Number of detected objects |
| avg_quality | Derived quality score |

## Architecture

This project follows the **Medallion Architecture**:

### Bronze Layer
- Raw data ingestion from FITS files  
- Feature extraction using Spark RDDs  
- Background subtraction and noise normalization  

### Silver Layer
- Data cleaning and validation  
- Pivot transformation for structured ML-ready dataset  
- Stored in **Parquet format**  

### Gold Layer
- Feature engineering and enrichment  
- Machine learning (K-Means clustering)  
- Final analytics-ready dataset  

---

## Processing Pipeline

1. **Data Ingestion**
   - Load FITS files using `sc.binaryFiles`
2. **Preprocessing**
   - Background subtraction (median normalization)
   - Statistical feature extraction (mean, std)
   - 3-sigma threshold for object detection
3. **Transformation**
   - Pivot image types into structured rows
4. **Modeling**
   - Feature scaling using `StandardScaler`
   - Clustering using **K-Means (k=3)**
5. **Enrichment**
   - Extract time, RA, Dec from object IDs

---

## Machine Learning Model

- Algorithm: **K-Means Clustering**
- Number of clusters: **3**
- Purpose:
  - Identify **transient objects**
  - Separate noise and anomalies  

### Performance
- Silhouette Score: **0.6842**
- Cluster distribution:
  - Cluster 0: Background detections  
  - Cluster 1: High-intensity transient candidates  
  - Cluster 2: Noise/artifacts
